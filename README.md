## 📖 About the Project

<div align="center">
<img src="https://readme-typing-svg.demolab.com?font=Fira+Code&size=16&duration=2000&pause=500&color=16A34A&center=true&vCenter=true&width=500&lines=One+Platform.+Full+Procurement+Control." alt="tagline" />
</div>

**VendorBridge** is a full-stack, real-time **Procurement and Vendor Management ERP** built to digitize and streamline procurement operations for organizations of any size.

From vendor onboarding to invoice generation — every step of the procurement lifecycle is handled through a single, unified platform with clean role-based workflows, real-time notifications, and automated document generation.

Whether you're a procurement officer raising an RFQ, a vendor submitting a quotation, a manager approving a deal, or an admin running analytics — VendorBridge gives every stakeholder exactly what they need, when they need it.

---

## ❌ The Problem

Most organizations today still manage procurement through:

- 📧 **Email chains** that go back and forth with vendors
- 📊 **Excel sheets** tracking quotations manually
- 🗂️ **Scattered documents** for purchase orders and invoices
- 🔍 **No visibility** into where a procurement request currently stands
- ⏳ **Delayed approvals** because there's no structured workflow
- 📉 **No analytics** on vendor performance or spending trends

This creates **confusion, delays, duplicate work, and costly mistakes** — especially when managing tens or hundreds of vendors simultaneously.

---

## ✅ Our Solution

VendorBridge brings the entire procurement lifecycle into **one centralized platform**:

| Traditional Way | VendorBridge Way |
|---|---|
| Email RFQs to vendors manually | Create structured RFQs and auto-notify vendors |
| Compare quotes in Excel | Side-by-side comparison with auto price highlighting |
| Approval via WhatsApp or calls | Structured approval workflow with audit trail |
| Manual PO creation in Word | Auto-generate POs with tax calculation |
| Email invoice as an attachment | Generate, download, and email invoice in one click |
| No visibility into procurement status | Real-time dashboard with live stats and charts |

## 🧩 Features

### 🔐 1. Authentication & Role Management
- Secure email + password login with **JWT authentication**
- Role-based access: Admin, Procurement Officer, Manager, Vendor
- Session persistence via localStorage
- Protected routes — every role sees only what they're allowed to

### 📊 2. Real-Time Dashboard
- **4 live stat cards**: Active RFQs, Pending Approvals, PO Value This Month, Overdue Invoices
- **Spending trends chart** — bar chart for last 6 months (Recharts)
- **Recent Purchase Orders** table with status badges
- Quick action buttons for common tasks
- Entire dashboard auto-refreshes via Socket.io — zero manual refresh needed

### 🏢 3. Vendor Management
- Full vendor registration with company name, GST number, category, contact details
- Vendor status tracking: Active / Inactive / Blacklisted
- Search by name, GST, category — filter by status
- Per-vendor history tab: all RFQs, quotations, and POs linked to that vendor
- Real-time notification to admins when a new vendor is registered

### 📋 4. RFQ Creation (Request for Quotation)
- 3-step RFQ creation wizard:
  - Step 1: Title, description, deadline
  - Step 2: Dynamic line items (product, quantity, unit, specifications)
  - Step 3: Multi-select vendor assignment
- Auto-generated RFQ number (e.g., `RFQ-2024-001`)
- Real-time notification sent to all assigned vendors upon RFQ creation
- RFQ status lifecycle: Open → Closed → Cancelled

### 💬 5. Vendor Quotation Submission
- Vendors see only their assigned RFQs
- Per-item pricing with auto-calculated totals
- Delivery timeline, notes, and editable before review starts
- Instant notification to procurement officer on submission

### ⚖️ 6. Quotation Comparison
- **Side-by-side comparison table** — all vendors' quotes in one view
- Lowest price per item **highlighted in green** automatically
- Grand total comparison with best-deal vendor highlighted
- Delivery timeline shown per vendor
- Vendor rating indicators
- "Select Vendor" button triggers the approval workflow

### ✅ 7. Approval Workflow
- Structured multi-step approval: Pending → Approved / Rejected
- Manager sees all pending approvals with full quotation context
- Approve or reject with remarks (remarks required on rejection)
- Approval timeline with timestamps
- Real-time notification to officer on every manager action
- Full audit trail maintained

### 📦 8. Purchase Order Generation
- Auto-generate PO from approved quotation
- Auto-generated PO number (e.g., `PO-2024-001`)
- Pre-filled vendor and item details from the quotation
- **18% GST auto-calculation** with grand total
- PO status lifecycle: Draft → Sent → Acknowledged → Completed

### 🧾 9. Invoice Generation & Delivery
- One-click invoice generation from any PO
- Auto-generated Invoice number (e.g., `INV-2024-001`)
- **Download as PDF** — professionally styled PDF via PDFKit
- **Print directly** from browser
- **Send via Email** — PDF attached and emailed to vendor via Nodemailer
- Mark invoice as Paid / Cancelled

### 🔔 10. Real-Time Notifications
- Bell icon with unread badge count
- Dropdown showing all notifications with timestamps
- Click any notification → navigate directly to that entity
- "Mark all as read" in one click
- All notifications persist in database — never lost

### 📜 11. Activity Logs
- Every create, update, approve, reject, generate action is logged
- Admin can view full audit trail with user, action, entity, timestamp
- Filter by user, entity type, date range
- Paginated timeline view

### 📈 12. Reports & Analytics
- **Monthly spending trends** — line chart for last 12 months
- **Spending by vendor category** — pie chart
- **Vendor performance table** — quotes submitted, POs won, total contract value
- **Top procurement stats** — avg approval time, total vendors, paid vs unpaid
- **CSV Export** for Vendors, RFQs, POs, and Invoices

## 🗄️ Database Schema

<details>
<summary><b>📂 Click to expand full schema (12 tables)</b></summary>

```sql
-- 1. Users
CREATE TABLE users (
  id              SERIAL PRIMARY KEY,
  name            VARCHAR(100) NOT NULL,
  email           VARCHAR(150) UNIQUE NOT NULL,
  password        VARCHAR(255) NOT NULL,
  role            VARCHAR(30) NOT NULL,   -- admin | procurement_officer | manager | vendor
  country         VARCHAR(100),
  phone           VARCHAR(20),
  photo_url       VARCHAR(255),
  is_active       BOOLEAN DEFAULT TRUE,
  created_at      TIMESTAMP DEFAULT NOW()
);

-- 2. Vendors
CREATE TABLE vendors (
  id              SERIAL PRIMARY KEY,
  user_id         INT REFERENCES users(id),
  company_name    VARCHAR(150) NOT NULL,
  category        VARCHAR(100),
  gst_number      VARCHAR(20) NOT NULL,
  contact_person  VARCHAR(100),
  email           VARCHAR(150) NOT NULL,
  phone           VARCHAR(20),
  address         TEXT,
  status          VARCHAR(20) DEFAULT 'active',  -- active | inactive | blacklisted
  created_at      TIMESTAMP DEFAULT NOW()
);

-- 3. RFQs (Request for Quotation)
CREATE TABLE rfqs (
  id              SERIAL PRIMARY KEY,
  rfq_number      VARCHAR(30) UNIQUE NOT NULL,
  title           VARCHAR(200) NOT NULL,
  description     TEXT,
  deadline        DATE,
  status          VARCHAR(20) DEFAULT 'open',  -- open | closed | cancelled
  created_by      INT REFERENCES users(id),
  created_at      TIMESTAMP DEFAULT NOW()
);

-- 4. RFQ Line Items
CREATE TABLE rfq_items (
  id              SERIAL PRIMARY KEY,
  rfq_id          INT REFERENCES rfqs(id) ON DELETE CASCADE,
  product_name    VARCHAR(200) NOT NULL,
  quantity        INT NOT NULL,
  unit            VARCHAR(30),
  specifications  TEXT
);

-- 5. RFQ Vendor Assignments
CREATE TABLE rfq_vendors (
  id              SERIAL PRIMARY KEY,
  rfq_id          INT REFERENCES rfqs(id),
  vendor_id       INT REFERENCES vendors(id),
  invited_at      TIMESTAMP DEFAULT NOW()
);

-- 6. Quotations
CREATE TABLE quotations (
  id              SERIAL PRIMARY KEY,
  rfq_id          INT REFERENCES rfqs(id),
  vendor_id       INT REFERENCES vendors(id),
  total_amount    DECIMAL(12,2),
  delivery_days   INT,
  notes           TEXT,
  status          VARCHAR(20) DEFAULT 'submitted',  -- submitted | under_review | selected | rejected
  submitted_at    TIMESTAMP DEFAULT NOW()
);

-- 7. Quotation Line Items
CREATE TABLE quotation_items (
  id              SERIAL PRIMARY KEY,
  quotation_id    INT REFERENCES quotations(id) ON DELETE CASCADE,
  rfq_item_id     INT REFERENCES rfq_items(id),
  unit_price      DECIMAL(12,2),
  total_price     DECIMAL(12,2)
);

-- 8. Approvals
CREATE TABLE approvals (
  id              SERIAL PRIMARY KEY,
  quotation_id    INT REFERENCES quotations(id),
  approver_id     INT REFERENCES users(id),
  status          VARCHAR(20) DEFAULT 'pending',  -- pending | approved | rejected
  remarks         TEXT,
  action_at       TIMESTAMP
);

-- 9. Purchase Orders
CREATE TABLE purchase_orders (
  id              SERIAL PRIMARY KEY,
  po_number       VARCHAR(30) UNIQUE NOT NULL,
  quotation_id    INT REFERENCES quotations(id),
  vendor_id       INT REFERENCES vendors(id),
  total_amount    DECIMAL(12,2),
  tax_amount      DECIMAL(12,2),
  grand_total     DECIMAL(12,2),
  status          VARCHAR(20) DEFAULT 'draft',  -- draft | sent | acknowledged | completed
  created_by      INT REFERENCES users(id),
  created_at      TIMESTAMP DEFAULT NOW()
);

-- 10. Invoices
CREATE TABLE invoices (
  id              SERIAL PRIMARY KEY,
  invoice_number  VARCHAR(30) UNIQUE NOT NULL,
  po_id           INT REFERENCES purchase_orders(id),
  amount          DECIMAL(12,2),
  tax_amount      DECIMAL(12,2),
  total_amount    DECIMAL(12,2),
  status          VARCHAR(20) DEFAULT 'unpaid',  -- unpaid | paid | cancelled
  due_date        DATE,
  created_at      TIMESTAMP DEFAULT NOW()
);

-- 11. Activity Logs
CREATE TABLE activity_logs (
  id              SERIAL PRIMARY KEY,
  user_id         INT REFERENCES users(id),
  action          VARCHAR(100),
  entity_type     VARCHAR(50),
  entity_id       INT,
  description     TEXT,
  created_at      TIMESTAMP DEFAULT NOW()
);

-- 12. Notifications
CREATE TABLE notifications (
  id              SERIAL PRIMARY KEY,
  user_id         INT REFERENCES users(id),
  title           VARCHAR(200),
  message         TEXT,
  is_read         BOOLEAN DEFAULT FALSE,
  created_at      TIMESTAMP DEFAULT NOW()
);
```

## ⚡ Real-Time Features

VendorBridge uses **Socket.io** for live, bidirectional communication between server and clients.

```
Event Flow Example — RFQ Creation:

Officer creates RFQ
      │
      ▼
Server: POST /api/rfqs
      │
      ├── Saves to DB
      ├── Logs to activity_logs
      ├── Creates notification records for each assigned vendor
      └── Emits "rfq_invite" to each vendor's socket room
                  │
                  ▼
      Vendor's browser (no refresh needed):
      ├── Bell badge count +1
      ├── Toast popup: "You've been invited to quote on [RFQ Title]"
      └── Notification appears in dropdown
```

**Full list of real-time events:**

| Event Name | Direction | Trigger | Recipients |
|---|---|---|---|
| `rfq_invite` | Server → Client | RFQ created | Assigned vendors |
| `quotation_submitted` | Server → Client | Vendor submits quote | Procurement Officers |
| `approval_request` | Server → Client | Officer selects vendor | All Managers |
| `approval_done` | Server → Client | Manager approves | Submitting Officer |
| `approval_rejected` | Server → Client | Manager rejects | Submitting Officer |
| `po_created` | Server → Client | PO generated | All Officers + Vendor |
| `invoice_created` | Server → Client | Invoice generated | Vendor |
| `new_notification` | Server → Client | Any above event | Target user |
| `dashboard_update` | Server → Broadcast | PO/Invoice status change | All connected users |

## 🚀 Getting Started

### Prerequisites

Make sure you have the following installed:

```bash
node -v      # v18 or higher
npm -v       # v9 or higher
psql --version  # PostgreSQL 14 or higher
```

### 1. Clone the Repository

```bash
git clone https://github.com/PDA-DP-Shop/vendorbridge.git
cd vendorbridge
```

### 2. Install Dependencies

```bash
# Install root dependencies
npm install

# Install server dependencies
cd server && npm install

# Install client dependencies
cd ../client && npm install
```

### 3. Set Up PostgreSQL Database

```bash
# Open PostgreSQL
psql -U postgres

# Create database
CREATE DATABASE vendorbridge;

# Connect to it
\c vendorbridge

# Run the schema
\i server/config/schema.sql
```

### 4. Configure Environment Variables

```bash
# In server/ folder
cp .env.example .env
# Then fill in your values (see Environment Variables section below)
```

### 5. Run the Application

```bash
# From root folder — runs both client and server together
npm run dev
```

This runs:
- **Server** at `http://localhost:5000`
- **Client** at `http://localhost:5173`

## 🔧 Environment Variables

Create a `.env` file inside the `server/` folder:

```env
# Server
PORT=5000
NODE_ENV=development

# Database
DB_HOST=localhost
DB_PORT=5432
DB_NAME=vendorbridge
DB_USER=postgres
DB_PASSWORD=your_password

# JWT
JWT_SECRET=your_super_secret_jwt_key_here
JWT_EXPIRES_IN=7d

# Email (Gmail SMTP)
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your_email@gmail.com
EMAIL_PASS=your_app_password

# Client URL (for CORS)
CLIENT_URL=http://localhost:5173
```

> **Note:** For Gmail, use an [App Password](https://support.google.com/accounts/answer/185833), not your regular Gmail password.
