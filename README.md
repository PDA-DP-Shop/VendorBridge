## 🌟 About The Project

Traditional procurement processes are often bogged down by scattered emails, messy spreadsheets, delayed approval chains, and untracked invoices. **VendorBridge** is built to bridge the gap between businesses and their suppliers. 

Inspired by modern ERP philosophies like **Odoo**, VendorBridge digitizes the entire supply chain lifecycle—from vendor onboarding and Request for Quotation (RFQ) creation to automated approval workflows, Purchase Orders (PO), and Invoice matching.

### 🎯 Key Objectives for the Hackathon
- **Eliminate Friction**: Centralize all supplier communication and bidding.
- **Automate Compliance**: Multi-level dynamic approval workflows.
- **Data-Driven Decisions**: Real-time analytical dashboard and immutable activity logs.

---

## ✨ Features

<img align="right" src="https://raw.githubusercontent.com/Tarikul-Islam-Anik/Animated-Fluent-Emojis/master/Emojis/Objects/Chart%20Increasing.png" alt="Chart" width="120" />

- **🏢 Vendor Directory:** Onboard, audit, and manage your external supply chain partners seamlessly.
- **📄 Sourcing & RFQs:** Generate detailed Requests for Quotations and invite multiple vendors to bid.
- **💬 Automated Bidding (Quotations):** Vendors have their own portal to submit and edit pricing proposals securely.
- **✅ Dynamic Approvals:** Multi-level workflow where POs and Quotes are routed to Team Leads, Managers, or the CFO based on monetary value.
- **📦 Purchase Orders (PO):** 1-click PO generation from approved quotations.
- **🧾 Ledger & Invoices:** Seamlessly match vendor invoices against purchase orders and track payment statuses (Paid/Unpaid).
- **📊 Analytics Dashboard:** Real-time metrics, spending patterns, and procurement cycle insights.
- **🔔 Real-Time Notifications:** Socket.io integration to alert users instantly when an RFQ is sent or an approval is required.

---

## 🛠 Tech Stack & Architecture

- **Frontend:** React (Vite), Tailwind CSS, Lucide React (Icons), React Router, Context API
- **Backend:** Node.js, Express.js
- **Database:** PostgreSQL (Relational Database for robust data integrity)
- **Real-Time Engine:** Socket.io (WebSockets)
- **Security:** JWT Authentication, Bcrypt Password Hashing, Role-Based Access Control (Admin, Procurement Officer, Manager, Vendor)

## 🚀 Getting Started

Follow these instructions to run VendorBridge on your local machine.

### Prerequisites
- Node.js (v16+)
- PostgreSQL installed and running

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/your-username/VendorBridge.git
   cd VendorBridge
   ```

2. **Setup Backend (Server)**
   ```bash
   cd server
   npm install
   ```
   *Create a `.env` file in the `/server` directory and add your Database URI, JWT Secret, and Port configuration.*
   ```bash
   npm run dev
   ```

3. **Setup Frontend (Client)**
   ```bash
   cd ../client
   npm install
   npm run dev
   ```

4. **Open your browser**
   Navigate to `http://localhost:5173` to see the application running.
